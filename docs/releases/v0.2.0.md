# Telegram proxy with DPI bypass

At this release, you can deploy a two-node Telegram proxy chain that bypasses
deep packet inspection (DPI). 🎉

Traffic from Telegram clients reaches a **relay server** (in an allowed zone),
gets wrapped in TLS via [GOST v3][] — so it looks like regular HTTPS to any DPI —
and is forwarded to a **gateway server** (outside the perimeter),
where the actual proxy services run on localhost.

## 🔑 Key features

### Three proxy modes

| Service       | Protocol | Auth                    | Notes                                    |
|:--------------|:---------|:------------------------|:-----------------------------------------|
| [mtg][]       | MTProto  | secret (auto-generated) | Native Telegram proxy, best performance  |
| [socks5][]    | SOCKS5   | username + password     | Universal, works for any app             |
| socks5-unsafe | SOCKS5   | none                    | Only accessible from the VPN server IP   |

All three are multiplexed through a **single TLS connection** between relay and gateway.

### Firewall hardening

Both servers are protected with UFW:

- **Relay**: MTProto and SOCKS5 ports open; `socks5-unsafe` port restricted to VPN server IP only
- **Gateway**: only the GOST TLS port is open, and only from the relay IP;
  all proxy services are bound to `127.0.0.1`

### Proxy auto-configuration (PAC)

Two PAC file templates for convenient routing of Telegram traffic through the no-auth SOCKS5 proxy:

- `proxy.pac.tpl` — for macOS system proxy settings
- `omega.pac.tpl` — for [Proxy SwitchyOmega 3][omega] browser extension

Both route `*.telegram.org`, `*.t.me`, and `*.cdn-telegram.org` through the proxy
and send everything else directly.

```bash
$ export RELAY_HOST=<relay_ip> SOCKS5_UNSAFE_PORT=<port>
$ make proxy
```

### Monitoring with Vector

[Vector][] agent deployed as a Docker container on every server.
Collects Docker container logs and host metrics (CPU, memory, disk, network),
ships them to [Grafana Cloud][] — Loki for logs, Prometheus for metrics.

```bash
$ make monitoring
```

## 🧩 Installation

```bash
$ git clone git@github.com:octomation/breakout.git
$ cd breakout && git checkout v0.2.0

# Fill in template variables — see README.md for the full list
$ cat ansible/hosts.tpl.ini \
  | sed "s/{{.VpnName}}/${VPN_NAME}/g" \
  | sed "s/{{.VpnHost}}/${VPN_HOST}/g" \
  | sed "s/{{.RelayName}}/${RELAY_NAME}/g" \
  | sed "s/{{.RelayHost}}/${RELAY_HOST}/g" \
  | sed "s/{{.GatewayName}}/${GATEWAY_NAME}/g" \
  | sed "s/{{.GatewayHost}}/${GATEWAY_HOST}/g" \
  | sed "s/{{.MTProtoPort}}/${MTPROTO_PORT}/g" \
  | sed "s/{{.SOCKS5Port}}/${SOCKS5_PORT}/g" \
  | sed "s/{{.SOCKS5UnsafePort}}/${SOCKS5_UNSAFE_PORT}/g" \
  | sed "s/{{.GostTLSPort}}/${GOST_TLS_PORT}/g" \
  | sed "s/{{.MTGDomain}}/${MTG_DOMAIN}/g" \
  | sed "s/{{.SOCKS5User}}/${SOCKS5_USER}/g" \
  | sed "s/{{.SOCKS5Password}}/${SOCKS5_PASSWORD}/g" \
  > ansible/hosts

$ make telegram
```

## 🔄 Changes for Outline VPN users

If you are upgrading from v0.1.0, the **docker** role has changed:

- **Docker SDK** updated from v5.0.0 to v7.1.0
- **docker-compose** standalone binary replaced by the official `docker-compose-plugin`
  (the old `/usr/local/bin/docker-compose` binary is removed automatically)
- Docker GPG key and apt repository setup updated to the current official method
  (keyrings-based instead of `apt-key`)
- Docker daemon now logs to **journald** with container name as tag
  (new `/etc/docker/daemon.json`; Docker will restart on first deploy)
- `break_system_packages: true` added for pip on newer Ubuntu distributions

These changes are applied automatically when you re-run `make setup`.
Outline VPN will continue to work — no manual intervention needed.
